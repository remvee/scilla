Scilla installation guide

Requirements

Several applications are needed to compile and run Scilla.

A list of compile-time dependencies:

Ant
Used to build and install Scilla. Available at the Jakarta project.
Servlet/JSP application server
Should work with any modern servlet engine. Tested with Tomcat and Jetty. Make sure the servlet API is available via your classpath before compilation.
JAI
Java Advanced Imaging API. Available at the Sun Microsystems. This imaging package can be omitted when you decided to use ImageMagick instead.

A list of run-time dependencies:

ImageMagick
A image manipulation suite. Available at the ImageMagick homepage.
Lame
A MP3 encoder used to recode MP3 files and encode WAV files to MP3. Available at the Lame homepage.
FFMpeg
A video format converter. Available at the FFMpeg homepage.

Examples installation

Extract the distributed zip-file and change directory to the newly created scilla directory. This directory contains a ant build file called build.xml. This file is used to compile all the Scilla Java classes, create a deployment hierarchy and a war file.

Build the examples war file by running ant. Here is an example:

Now copy the war file to the application server deployment directory (typically called webapps). Tomcat needs no further configuration, upon restart it will recognize the new application and start it. Jetty needs to be told a new application is added but that's out of the scope of this document.

Now restart the server and point your browser to your scilla examples site; something like: http://localhost:8080/scilla/. A page with links to the examples should be visible now. Enjoy..

Usage

Please look at the included build.xml files and web.xml file for details on installation and look at the examples JSPs (and HTML source as generated by the JSPs) for pointers on the usage of the converters. All converters are documented using javadoc.
